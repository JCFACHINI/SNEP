Ramais - Incluir/Alterar

Esta rotina permite o Cadastro/Alteração de uma Ramal no Sistema. Os Ramais definem os membros dos sistemas Asterisk e SNEP, ou seja, servem para identificar um membro no sistema de telefonia e para acessar este sistema de Gerenciamento.

Para o cadastro de Ramais é necessário que se ajuste, em Configurações >> Parâmetros, os valores dos campos referentes a Tecnologias de Canais conforme for o caso: SIP,IAX2 ou KHOMP

  • Ramal: Define o código do que identificará o Ramal. O Sistema sempre irá sugerir o número seguinte ao maior valor já cadastrado. Este código também será o LOGIN para o SNEP.
  • Nome Completo: Define um nome para o Ramal.
  • Senha: Define uma senha para o Ramal, que também será a senha para o LOGIN no SNEP.
  • E-Mail: Define um e-mail para o Ramal.
  • Vínculos: Define uma lista de OUTROS Ramais sobre os quais o Ramal em questão tem acesso às informações. Se NENHUM vínculo for definido, o Ramal em questão terá acesso completo aos relatórios de todos os Ramais. isto é válido para quando o Ramal em questão efetuar LOGIn no SNEP.
  • Limite de Chamadas de Entrada: Define um número que irá limitar o número de chamadas que o Ramal pode receber.
  • Limite de Chamadas de Saída: Define um número que irá limitar o número de chamadas que o Ramal pode fazer.
  • Usa Voicemail : Define se o Ramal vai ou não utilizar Correio de Voz. Em caso positivo, uma senha EXCLUSIVAMENTE NUMéRICA deverá ser informada.
  • Usa Autenticação (cadeado): Define se o Ramal irá usar ou não uma senha para poder efetuar ligações. Em caso positivo, uma senha EXCLUSIVAMENTE NUMéRICA deverá ser informada.
  • Grupo de Ramais:Define o grupo de Ramais qual o ramal pertence

  • OPÇÕES AVANÇADAS

    • Grupo de Captura: Grupo de Captura do qual o Ramal fará parte. Ramais em um mesmo grupo de captura permitem-se capturar ligaços uns dos outros (Cadastros >> Grupos de Captura).
    • Atravessar NAT: Define se a conexão do Ramal irá ou não através um NAT.
    • Caixa de Mensagens: Define o número da Caixa de mensagens
    • Controle de Qualidade: Define se haverá ou não controle de qualidade nas ligações do Ramal.
    • Modo de DTMF: Define o modo de DTMF do Ramal (Dual-Tone Multi-Frequency = Tons de multi frequências.
    • CODECS Permitidos: Define quais os CODECS permitidos e preferidos para o Ramal, na ordem que for escolhido.
    • Canais: Define qual tecnologia de canal o ramal irá usar para fazer/receber ligações.
      • Virtual: ramais com interfaces virtuais são mapeados em um tronco. Ao selecionar um tronco para a interface virtual do ramal, a ligação para ele será feita discando através do tronco para o numero do ramal.
    • Visualizar Status...: Define de quais filas o Ramal, quando LOGADO no SNEP poderá acompanhar o Status (Status >> Filas)
